                <p>يعتمد المشروع على جمع البيانات وتنظيمها وتحليلها باستخدام أدوات وتقنيات تحليل البيانات، بهدف الوصول إلى مؤشرات تساعد أصحاب المشاريع الصغيرة على اتخاذ قرارات أفضل، وتحسين التخطيط والمتابعة، وزيادة فرص نجاح المشاريع. من خلال هذا التحليل، نسعى إلى تقديم رؤى قابلة للتنفيذ تدعم رواد الأعمال في رحلتهم نحو النمو والاستدامة.</p>

                <section class="analysis-section">
                    <h2>الفصل الأول: مدخل إلى علم تحليل البيانات</h2>
                    <h3>1.1 التعريف وأهمية تحليل البيانات</h3>
                    <p>تحليل البيانات هو عملية فحص البيانات وتنقيتها وتحويلها بهدف استخلاص معلومات مفيدة لدعم اتخاذ القرارات. مع التطور الرقمي، أصبحت البيانات هي النفط الجديد، وتحليلها هو الطريقة الأمثل لاستخراج قيمتها. في المشاريع الصغيرة، يساعد تحليل البيانات على تحسين الكفاءة التشغيلية، فهم العملاء، التنبؤ بالمبيعات، تقليل التكاليف، وزيادة الأرباح.</p>
                    <div class="stat-highlight">أكثر من 70% من المشاريع الصغيرة التي تتبنى تحليل البيانات تحقق نمواً مضاعفاً خلال أول عامين.</div>
                    
                    <h3>1.2 مراحل تطور تحليل البيانات عبر العقود</h3>
                    <ul>
                        <li><strong>الخمسينيات - السبعينيات:</strong> التحليل اليدوي باستخدام السجلات الورقية والآلات الحاسبة البدائية.</li>
                        <li><strong>الثمانينيات:</strong> ظهور الحواسيب الشخصية وبرامج جداول البيانات مثل VisiCalc و Lotus 1-2-3.</li>
                        <li><strong>التسعينيات:</strong> انتشار قواعد البيانات العلائقية (SQL) وأنظمة تخطيط الموارد (ERP).</li>
                        <li><strong>الألفية الجديدة:</strong> ظهور البيانات الضخمة (Big Data) وتقنيات التخزين الموزعة (Hadoop).</li>
                        <li><strong>2010 - 2020:</strong> ثورة الذكاء الاصطناعي وتعلم الآلة وتحليل البيانات التنبؤية.</li>
                        <li><strong>2020 - الآن:</strong> دمج تحليل البيانات مع إنترنت الأشياء والحوسبة السحابية والتحليل الفوري.</li>
                    </ul>

                    <h3>1.3 الأبعاد الأربعة لتحليل البيانات</h3>
                    <ul>
                        <li><strong>التحليل الوصفي (Descriptive):</strong> يركز على وصف ما حدث بالفعل (مثل تقارير المبيعات الشهرية).</li>
                        <li><strong>التحليل التشخيصي (Diagnostic):</strong> يبحث في أسباب حدوث الأمور (لماذا انخفضت المبيعات؟).</li>
                        <li><strong>التحليل التنبؤي (Predictive):</strong> يستخدم النماذج الإحصائية لتوقع ما سيحدث (توقعات المبيعات للربع القادم).</li>
                        <li><strong>التحليل التوجيهي (Prescriptive):</strong> يقدم توصيات وإجراءات محددة (ما الإجراءات التي يجب اتخاذها لتحقيق الأهداف).</li>
                    </ul>
                </section>

                        <div class="importance-item"><h3>10. جذب الاستثمارات</h3><p>المستثمرون يفضلون المشاريع التي تعتمد على البيانات ولديها رؤية واضحة مدعومة بالأرقام، مما يزيد فرص الحصول على التمويل.</p></div>

                <section class="analysis-section">
                    <h2>الفصل الثالث: مصادر البيانات وأنواعها في المشاريع الصغيرة</h2>
                    <h3>3.1 البيانات المالية والمحاسبية</h3>
                    <p>تشمل الإيرادات، المصروفات، الأرباح، التدفقات النقدية، المبيعات اليومية، الفواتير، هوامش الربح، عائد الاستثمار، تكلفة البضاعة المباعة. هذه البيانات هي أساس أي تحليل مالي.</p>
                    
                    <h3>3.2 بيانات العملاء والتسويق</h3>
                    <p>المعلومات الديموغرافية (العمر، الجنس، الموقع)، سلوك الشراء، تفضيلات المنتجات، قنوات الوصول، تفاعلات وسائل التواصل الاجتماعي، نتائج الاستبيانات، شكاوى العملاء، معدلات التحويل، تكلفة اكتساب العميل.</p>
                    
                    <h3>3.3 بيانات العمليات والإنتاج</h3>
                    <p>كفاءة الإنتاج، أوقات التصنيع، جودة المنتج (معدلات العيوب)، إنتاجية الموظفين، سرعة دوران المخزون، أوقات التسليم، استخدام الموارد.</p>
                    
                    <h3>3.4 بيانات الموارد البشرية</h3>
                    <p>أداء الموظفين، معدلات الغياب، دوران العمالة، تكاليف التدريب، رضا الموظفين، ساعات العمل الإضافية.</p>
                    
                    <h3>3.5 البيانات الخارجية</h3>
                    <p>اتجاهات السوق، تحليل المنافسين، المؤشرات الاقتصادية، البيانات السكانية، تقارير الصناعة، الطقس (لبعض المشاريع).</p>
                    
                    <h3>3.6 مصادر جمع البيانات</h3>
                    <ul>
                        <li><strong>نقاط البيع (POS):</strong> لتسجيل المبيعات الفورية.</li>
                        <li><strong>أنظمة إدارة العلاقات مع العملاء (CRM):</strong> لتتبع تفاعلات العملاء.</li>
                        <li><strong>أنظمة تخطيط الموارد (ERP):</strong> لدمج بيانات الإدارة المالية والتشغيلية.</li>
                        <li><strong>أدوات تحليلات الويب (Google Analytics):</strong> لفهم سلوك زوار الموقع الإلكتروني.</li>
                        <li><strong>منصات التواصل الاجتماعي:</strong> لقياس التفاعل والوصول.</li>
                        <li><strong>الاستبيانات واستمارات التغذية الراجعة.</strong></li>
                        <li><strong>الأجهزة الذكية وأجهزة الاستشعار (لإنترنت الأشياء).</strong></li>
                    </ul>

                <section class="analysis-section">
                    <h2>الفصل الخامس: المنهجية العلمية لتطبيق تحليل البيانات (خطوة بخطوة)</h2>
                    <h3>5.1 تحديد أهداف العمل (Business Understanding)</h3>
                    <p>تبدأ أي عملية تحليل بتحديد الأهداف بوضوح وفقاً لمعايير SMART: محددة (Specific)، قابلة للقياس (Measurable)، قابلة للتحقيق (Achievable)، مرتبطة بالمشروع (Relevant)، محددة زمنياً (Time-bound). مثال: "زيادة المبيعات عبر الإنترنت بنسبة 20% خلال الربع القادم".</p>
                    
                    <h3>5.2 فهم البيانات (Data Understanding)</h3>
                    <p>جمع البيانات الأولية من المصادر المحددة، واستكشافها لفهم خصائصها، وتحديد جودتها، واكتشاف الأنماط الأولية. يشمل ذلك الإحصائيات الوصفية والتصورات الأولية.</p>
                    
                    <h3>5.3 إعداد وتنظيف البيانات (Data Preparation)</h3>
                    <p>أهم وأكثر خطوة تستهلك وقتاً (تصل إلى 80% من وقت المشروع). تتضمن: تنظيف البيانات (معالجة القيم المفقودة، إزالة التكرارات، تصحيح الأخطاء)، تحويل البيانات (تطبيع، توحيد مقاييس)، إنشاء متغيرات جديدة (Feature Engineering)، دمج البيانات من مصادر مختلفة.</p>
                    <div class="quote-box">"البيانات النظيفة هي أساس أي تحليل ناجح. القمامة داخلاً = قمامة خارجاً (Garbage In, Garbage Out)."</div>
                    
                    <h3>5.4 النمذجة (Modeling)</h3>
                    <p>اختيار وتطبيق تقنيات التحليل المناسبة (إحصائية، تعلم آلة، استكشافية) على البيانات المجهزة. يتم تدريب النماذج واختبارها وتحسين معلماتها للحصول على أفضل أداء.</p>
                    
                    <h3>5.5 تقييم النماذج (Evaluation)</h3>
                    <p>تقييم نتائج النماذج للتأكد من أنها تحقق أهداف العمل المحددة، وليست مجرد نتائج إحصائية جيدة. يتم التحقق من دقة النموذج وقوته التفسيرية.</p>
                    
                    <h3>5.6 نشر النتائج (Deployment)</h3>
                    <p>تقديم نتائج التحليل بشكل مفهوم ومؤثر لأصحاب القرار. قد يشمل ذلك تقارير مكتوبة، عروض تقديمية، لوحات معلومات تفاعلية، أو حتى دمج النموذج في نظام تشغيلي. المتابعة وقياس أثر التوصيات جزء أساسي من هذه المرحلة.</p>
                </section>

                <section class="analysis-section">
                    <h2>الفصل السادس: مؤشرات الأداء الرئيسية (KPIs) بالتفصيل الممل</h2>
                    <div class="stats-grid-detailed">
                        <div class="stat-item-detailed"><h4>إجمالي الإيرادات (Total Revenue)</h4><p>مجموع الدخل من المبيعات. يقيس حجم الأعمال. يُفضل متابعته يومياً/شهرياً.</p></div>
                        <div class="stat-item-detailed"><h4>صافي الربح (Net Profit)</h4><p>الإيرادات - جميع المصروفات. المؤشر الحقيقي لربحية المشروع. يستهدف عادة 10-20% من الإيرادات.</p></div>
                        <div class="stat-item-detailed"><h4>هامش الربح الإجمالي (Gross Profit Margin)</h4><p>(الإيرادات - تكلفة البضاعة) / الإيرادات. يقيس كفاءة الإنتاج والتسعير. النسبة المثالية تختلف حسب القطاع (غالباً >40%).</p></div>
                        <div class="stat-item-detailed"><h4>عائد الاستثمار (ROI)</h4><p>(الربح - تكلفة الاستثمار) / تكلفة الاستثمار. يقيس كفاءة استثمار معين. يستهدف عادة >20%.</p></div>
                        <div class="stat-item-detailed"><h4>تكلفة اكتساب العميل (CAC)</h4><p>إجمالي مصاريف التسويق والمبيعات / عدد العملاء الجدد. كلما كان أقل كان أفضل، ولكن يجب مقارنته بالقيمة الدائمة للعميل.</p></div>
                        <div class="stat-item-detailed"><h4>القيمة الدائمة للعميل (CLV)</h4><p>متوسط قيمة الشراء × متوسط عدد المشتريات في السنة × متوسط عدد سنوات الاحتفاظ بالعميل. يحدد مقدار ما يمكن استثماره لاكتساب عميل جديد.</p></div>
                        <div class="stat-item-detailed"><h4>معدل الاحتفاظ بالعملاء (Retention Rate)</h4><p>(عدد العملاء في نهاية الفترة - العملاء الجدد) / عدد العملاء في بداية الفترة. مؤشر على ولاء العملاء وجودة الخدمة/المنتج.</p></div>
                        <div class="stat-item-detailed"><h4>معدل التحويل (Conversion Rate)</h4><p>عدد الزوار الذين نفذوا الإجراء المطلوب (شراء، اشتراك) / إجمالي عدد الزوار. يقيس فعالية الموقع/الحملة التسويقية.</p></div>
                        <div class="stat-item-detailed"><h4>معدل دوران المخزون (Inventory Turnover)</h4><p>تكلفة البضاعة المباعة / متوسط قيمة المخزون. يقيس كفاءة إدارة المخزون. الارتفاع قد يعني مبيعات جيدة أو نقص في المخزون، والانخفاض قد يعني زيادة في المخزون أو ضعف مبيعات.</p></div>
                        <div class="stat-item-detailed"><h4>صافي نقاط الترويج (NPS)</h4><p>مقياس لرضا العملاء وولائهم من خلال سؤال: "ما مدى احتمالية أن توصي بنا لصديق؟". الدرجات من -100 إلى +100، والدرجات الموجبة تعتبر جيدة.</p></div>
                        <div class="stat-item-detailed"><h4>متوسط قيمة الطلب (AOV)</h4><p>إجمالي الإيرادات / عدد الطلبات. يمكن زيادتها عبر البيع الإضافي والتجميعي.</p></div>
                        <div class="stat-item-detailed"><h4>إنتاجية الموظف (Employee Productivity)</h4><p>الإيرادات / عدد الموظفين، أو عدد الوحدات المنتجة / عدد ساعات العمل. يقيس كفاءة القوى العاملة.</p></div>
                        <div class="stat-item-detailed"><h4>نسبة العيوب (Defect Rate)</h4><p>عدد المنتجات المعيبة / إجمالي المنتجات المنتجة. مؤشر رئيسي لجودة الإنتاج.</p></div>
                        <div class="stat-item-detailed"><h4>نسبة السيولة السريعة (Quick Ratio)</h4><p>(الأصول المتداولة - المخزون) / الخصوم المتداولة. يقيس قدرة المشروع على سداد التزاماته قصيرة الأجل.</p></div>
                    </div>
                </section>

                        <h3>دراسة حالة (1): متجر إلكتروني للإكسسوارات النسائية</h3>

                        <p><strong>النتائج بعد 4 أشهر:</strong> انخفاض CAC إلى 290 جنيه (▼36%)، ارتفاع معدل التحويل إلى 2.9% (▲142%)، انخفاض نسبة التخلي عن السلة إلى 60% (▼23%)، زيادة الإيرادات بنسبة 95%.</p>

                        <p><strong>المشكلة:</strong> انخفاض المبيعات في أيام منتصف الأسبوع، وارتفاع هدر المواد الغذائية إلى 15%، وشكاوى من بطء الخدمة في أوقات الذروة.</p>
                        <p><strong>التحليل:</strong> تحليل بيانات المبيعات (نظام نقاط البيع) كشف عن انخفاض مبيعات أيام الأحد والإثنين. تحليل المخزون أظهر أن هدر المواد يتركز في صنفين رئيسيين. مراقبة أوقات الخدمة في أوقات الذروة كشفت عن اختناق في المطبخ بسبب طلب عالٍ على وجبة معينة.</p>
                        <p><strong>الإجراءات:</strong> تقديم عروض خاصة (خصومات، وجبات غداء) أيام الأحد والإثنين. تعديل قائمة الطعام لتقليل المكونات القابلة للتلف، وتدريب الطهاة على إعادة استخدامها. تعديل تجهيزات المطبخ قبل أوقات الذروة، وإضافة خيار الطلب المسبق.</p>
                        <p><strong>النتائج بعد 3 أشهر:</strong> زيادة مبيعات أيام الركود بنسبة 45%، تقليل هدر المواد إلى 6% (▼60%)، انخفاض وقت انتظار العملاء في الذروة بنسبة 35%، ارتفاع هامش الربح الإجمالي من 15% إلى 24%.</p>

                        <p><strong>الإجراءات:</strong> تطوير عملية تأهيل (Onboarding) تفاعلية تركز على شرح الميزات الأساسية والتكاملات. إطلاق حملات بريد إلكتروني تلقائية لتشجيع استخدام هذه الميزات. تخصيص فريق دعم للتواصل مع العملاء الجدد في الأسبوع الأول.</p>

                <section class="analysis-section">
                    <h2>الفصل التاسع: تقنيات متقدمة في تحليل البيانات</h2>
                    <h3>9.1 التحليل العنقودي (Clustering)</h3>
                    <p>تقنية تقسيم العملاء أو المنتجات إلى مجموعات متشابهة بناءً على خصائص معينة. يساعد في التخصيص والتسويق المستهدف. (مثال: تقسيم العملاء إلى وفّير، متوسط، اقتصادي).</p>
                    
                    <h3>9.2 تحليل الانحدار (Regression Analysis)</h3>
                    <p>فهم العلاقة بين متغير تابع (مثل المبيعات) ومتغيرات مستقلة (مثل الإنفاق الإعلاني، الموسم). يستخدم للتنبؤ وتحديد العوامل الأكثر تأثيراً.</p>
                    
                    <h3>9.3 تحليل السلاسل الزمنية (Time Series Analysis)</h3>
                    <p>تحليل بيانات مرتبة زمنياً لتحديد الأنماط (موسمية، اتجاهات) والتنبؤ بالقيم المستقبلية. أساسي لتوقع المبيعات والمخزون.</p>
                    
                    <h3>9.4 تحليل سلة السوق (Market Basket Analysis)</h3>
                    <p>تقنية اكتشاف العلاقات بين المنتجات التي يتم شراؤها معاً. تستخدم لتوصيات المنتجات وتصميم العروض. (مثال: من اشترى حفاضات اشترى أيضاً مناديل مبللة).</p>
                    
                    <h3>9.5 تحليل المشاعر (Sentiment Analysis)</h3>
                    <p>استخدام معالجة اللغة الطبيعية لتحليل آراء العملاء على وسائل التواصل الاجتماعي والمراجعات، وتحديد ما إذا كانت إيجابية، سلبية، أو محايدة.</p>
                    
                    <h3>9.6 النمذجة التنبؤية (Predictive Modeling)</h3>
                    <p>استخدام تقنيات تعلم الآلة (مثل أشجار القرار، الغابات العشوائية، الشبكات العصبية) لبناء نماذج تتنبأ بسلوك العملاء، احتمالية توقفهم، أو تقدير القيمة الدائمة لهم.</p>
                </section>

        function downloadPDF() {
            try {
                const activePage = document.querySelector('.page.active');
                if (!activePage) return;
                
                const opt = {
                    margin: [0.3, 0.3, 0.3, 0.3],
                    filename: 'تقرير_مشروع_التخرج.pdf',
                    image: { type: 'jpeg', quality: 0.95 },
                    html2canvas: { scale: 1.5, useCORS: true, logging: false },
                    jsPDF: { unit: 'in', format: 'a4', orientation: 'portrait' }
                };
                alert('جاري تحميل ملف PDF للصفحة الحالية...');
                html2pdf().from(activePage).set(opt).save();
            } catch (error) {
                alert('حدث خطأ أثناء تحميل PDF');
            }
        }

        function downloadPPT() {
            try {
                const activePage = document.querySelector('.page.active');
                if (!activePage) return;
                
                const clonePage = activePage.cloneNode(true);
                const buttons = clonePage.querySelector('.download-buttons');
                if (buttons) buttons.remove();
                
                const style = document.createElement('style');
                style.textContent = `
                    body { font-family: 'Cairo', sans-serif; margin: 0; padding: 20px; direction: rtl; background: white; }
                    .page { display: block !important; box-shadow: none; padding: 30px; }
                    .download-buttons, .navbar, footer { display: none; }
                `;
                
                const htmlContent = `
                    <!DOCTYPE html>
                    <html dir="rtl">
                    <head><meta charset="UTF-8"><title>تقرير مشروع التخرج</title>
                    <link href="https://fonts.googleapis.com/css2?family=Cairo:wght@400;600;700;800&display=swap" rel="stylesheet">
                    <style>${document.querySelector('style').innerHTML}${style.textContent}</style></head>
                    <body>${clonePage.outerHTML}</body></html>
                `;
                
                const blob = new Blob([htmlContent], { type: 'text/html' });
                const url = URL.createObjectURL(blob);
                const a = document.createElement('a');
                a.href = url;
                a.download = 'تقرير_مشروع_التخرج.html';
                document.body.appendChild(a);
                a.click();
                document.body.removeChild(a);
                URL.revokeObjectURL(url);
                alert('تم التحميل. افتح الملف في PowerPoint.');
            } catch (error) {
                alert('حدث خطأ');
            }
        }
